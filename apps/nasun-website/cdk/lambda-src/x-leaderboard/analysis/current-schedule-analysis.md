## 📊 API 사용량 분석

일일 제한: 1,667 requests/day
15분 제한: 15 requests/15분 (user context endpoints)

### 현재 예상 API 호출량

#### 1. V1 시스템 (기존)
- 인게이지먼트 수집: ~46회/일
- 사용률: 2.8% (46/1,667)

#### 2. V2 시스템 (누적)
- 누적 데이터 수집: ~50회/일 (예상)
- 사용률: 3.0% (50/1,667)

#### 3. 리트윗 보너스 (Phase 1)
- 타겟 리트윗 타임라인: 1회/일
- 원본 트윗 조회: 최대 100회/일
- **합계**: 101회/일
- 사용률: 6.1% (101/1,667)

### 🚨 총 API 사용량 계산
```
V1: 46회 + V2: 50회 + 리트윗보너스: 101회 = 197회/일
총 사용률: 11.8% (197/1,667)
```

## ⚠️ 식별된 문제점

### 1. 시간대 집중 문제
- **00:00-01:30 UTC**: 3개 시스템이 연속 실행
- **04:30 UTC**: 리트윗 보너스만 단독 실행
- **나머지 시간**: API 미사용 (비효율적)

### 2. Rate Limit 위험성
- **15분 제한**: 리트윗 보너스에서 최대 100개 트윗 조회 시 위험
- **집중 실행**: 짧은 시간에 많은 API 호출

### 3. 의존성 문제
- V1과 V2가 독립적으로 실행되지만 같은 시간대 집중
- 리트윗 보너스는 타겟 계정의 활동 패턴과 무관한 시간 실행
