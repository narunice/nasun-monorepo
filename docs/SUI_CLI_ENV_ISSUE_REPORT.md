# SUI CLI Environment Recognition Issue Report

## 1. Issue Overview
**Date:** January 1, 2026
**Target System:** Nasun Devnet (SUI Fork)
**CLI Version:** `sui 1.63.0-775f237e3ec2`
**Project Path:** `apps/pado/contracts`

**Problem Statement:**
When attempting to build or upgrade a Move package using `sui move build` or `sui client upgrade`, the operation fails with the following error, despite the environment being correctly configured in `Move.toml`:

> Your active environment `nasun-devnet` is not present in `Move.toml`, so you cannot publish to `nasun-devnet`.

This occurs even when the `[environments]` section explicitly contains the active environment and its Chain ID.

## 4. Hypothesis & Conclusion

**Hypothesis:**
The SUI CLI (v1.63.0) likely contains a bug in its `Move.toml` parsing logic, specifically regarding the validation of the `[environments]` section against the active CLI environment. It appears to ignore the section entirely or fails to match custom environment names/IDs correctly, enforcing a strict check that cannot be satisfied by standard configuration.

**Current Blockers:**
1.  **Upgrade Impossible via CLI:** The `sui client upgrade` command strictly enforces the check, preventing standard upgrades.
2.  **Build Blocked:** Standard `sui move build` is also blocked.

**Workarounds:**
*   **New Deployment:** `sui client test-publish` can be used to deploy a *new* package instance (addresses must be reset to `0x0`).
*   **Manual PTB Upgrade:** Theoretically possible by using bytecode generated from `test-publish` and constructing a `PackageUpgrade` transaction manually via `sui client ptb`, but high complexity and risk.

## 5. Recommendation for Claude
This context suggests that standard CLI paths are broken for this specific version/environment combination. Focus on:
1.  Investigating if a manual **Programmable Transaction Block (PTB)** construction can perform the upgrade using the bytecode generated by `test-publish`.
2.  Alternatively, advising on a migration strategy to a **new deployment** if preserving the package address is not critical.
