## 앱별 역할

### Pado = 금융 슈퍼앱 (금융센터)

하나의 앱에서 여러 금융 서비스를 제공하는 올인원 플랫폼.

```
Pado 금융센터
├── DEX (거래소) ← DeepBook V3 기반 호가창 거래
│   ├── Spot Trading (현물 거래)
│   ├── Orderbook (호가창)
│   └── Trade History (거래 기록)
├── Perp (무기한 선물) ← 레버리지 거래
│   ├── Long/Short (매수/매도)
│   ├── Leverage (1x ~ 20x)
│   └── Liquidation Engine (자동 청산)
├── Prediction Market (예측 시장) ← "비트코인이 10만 달러 넘을까?"
│   ├── 시장 생성 (관리자)
│   ├── YES/NO 토큰 매매
│   └── 결과 확정 + 정산
├── Lottery (복권) ← 온체인 추첨
│   ├── 티켓 구매
│   ├── 랜덤 추첨 (Sui Random)
│   └── 당첨금 수령
├── Lending (대출) ← 자산 담보 대출
├── Portfolio (포트폴리오) ← 내 자산/수익 분석
├── Leaderboard (리더보드) ← 트레이더 순위
├── Chat (채팅) ← 실시간 글로벌 채팅
└── Badges (뱃지) ← 활동 보상 시스템
```

**핵심 기술**: DeepBook V3 (CLOB), Unified Margin, Oracle, LP Bot

### Baram = AI 안전 거래소

AI 서비스를 안전하게(프라이버시 보장) 거래하는 플랫폼.

```
사용자의 AI 요청
       ↓
  [암호화] ← 내용을 아무도 못 봄
       ↓
  [TEE 금고] ← AWS Nitro Enclave
  │ 1. 복호화
  │ 2. AI 모델 실행
  │ 3. 결과 암호화
       ↓
  [에스크로 정산] ← 스마트 컨트랙트가 자동 정산
       ↓
  사용자에게 결과 전달
```

**핵심 기술**: AWS Nitro Enclave (TEE), Move Escrow, Budget Delegation

### Network Explorer = 거래 추적 센터

블록체인에서 일어나는 모든 활동을 검색하고 확인하는 도구.
은행 앱에서 거래 내역을 조회하는 것과 비슷.

- 트랜잭션 검색 (거래 내역 조회)
- 지갑 주소 조회 (계좌 조회)
- 블록/체크포인트 탐색 (장부 열람)
- 밸리데이터 정보 (네트워크 운영자 현황)

### nasun-website = 시청 (공식 웹사이트)

프로젝트의 공식 창구. 커뮤니티 관리와 거버넌스.

- 리더보드 V3 (커뮤니티 기여도 순위)
- 거버넌스 (투표/제안)
- NFT 이벤트 (커뮤니티 참여 보상)
- 회원 관리 (Twitter/Google 인증, 지갑 연동)

### gensol-website = 연구소 (GenSol/Spectra)

AI 관련 비전과 기술을 소개하는 웹사이트.

## 공유 패키지: 모든 건물의 공용 시설

```
packages/
├── wallet/          ← 공용 출입카드 시스템
│   ├── 지갑 생성/잠금/해제
│   ├── 잔액 조회
│   ├── 거래 전송
│   └── zkLogin (Google 로그인)
│
├── wallet-ui/       ← 출입카드 리더기 (UI 컴포넌트)
│   ├── 로그인 화면
│   ├── 잔액 표시
│   └── 토큰 전송 폼
│
├── devnet-config/   ← 건물 주소록
│   └── 모든 스마트 컨트랙트의 주소를 한 곳에서 관리
│
├── tsconfig/        ← 건축 규격서 (TypeScript 설정 공유)
└── tailwind-config/ ← 인테리어 가이드 (브랜드 색상 통일)
```

**왜 공유하나?**: 각 앱마다 지갑 코드를 따로 만들면 유지보수가 5배.
하나를 만들어서 모든 앱이 가져다 쓰면 한 번만 고치면 됨.

## 배포 파이프라인: 코드가 사용자에게 도달하는 과정

```
개발자 컴퓨터 (코드 작성)
       ↓
  [pnpm build] ← 코드를 배포용으로 포장
       ↓
  [rsync] ← 변경된 파일만 서버로 전송
       ↓
  EC2 서버 (Amazon 임대 컴퓨터)
       ↓
  [Nginx] ← 사용자 요청을 올바른 앱으로 안내
       ↓
  사용자 브라우저
```

**현재 상태**: 모든 배포가 수동 (CI/CD 없음).
개발자가 직접 스크립트를 실행해서 배포.

| 서비스 | URL | 배포 스크립트 |
|--------|-----|-------------|
| Pado | pado.finance | deploy-pado-production.sh |
| nasun-website | nasun.io | deploy-nasun-website-production.sh |
| Explorer | explorer.nasun.io | deploy-network-explorer-production.sh |
| LP Bot | EC2 + PM2 | deploy-pado-bots.sh |

## 트레이딩 데이터 흐름 (Pado)

사용자가 거래 버튼을 누르면 일어나는 일:

```
사용자: "BTC를 100 USDC에 사고 싶다"
       ↓
  [PerpOrderForm] ← 주문 폼 UI
       ↓
  [useTrading] ← 주문 데이터 준비
       ↓
  [Transaction 빌드] ← Move 함수 호출 준비
       ↓
  [지갑 서명] ← 개인키로 "내가 이 거래를 승인합니다" 서명
       ↓
  [블록체인 전송] ← Nasun Devnet에 트랜잭션 제출
       ↓
  [DeepBook V3] ← 온체인 호가창에서 주문 매칭
       ↓
  [이벤트 발생] ← "주문 체결됨" 이벤트
       ↓
  [useOrderFillNotifier] ← 체결 알림 (토스트 + 소리)
       ↓
  사용자: "주문이 체결되었습니다!"
```

## zkLogin 인증 흐름

사용자가 Google로 로그인할 때 일어나는 일:

```
사용자: "Google로 로그인" 클릭
       ↓
  [Google OAuth] ← Google에서 "이 사람 맞습니다" 확인 (JWT 토큰)
       ↓
  [Salt API] ← 이 사용자 고유의 "비밀 양념" 가져옴
       ↓
  [임시 키 생성] ← 이번 세션에서만 쓸 일회용 도장 생성
       ↓
  [ZK Proof 생성] ← "나는 Google 인증을 받은 사람이다"를
  │                  개인정보 없이 수학적으로 증명
       ↓
  [블록체인 지갑 연결] ← ZK Proof를 사용해 거래 서명 가능
       ↓
  사용자: 지갑 연결 완료! (Google에는 지갑 정보가 안 보임)
```

## LP Bot 자동 거래 흐름

사람 대신 자동으로 호가를 채워주는 봇:

```
[10초마다 반복]
       ↓
  바이낸스에서 BTC 실시간 가격 가져옴
       ↓
  현재 가격 기준으로 주문 계산:
  ├── 현재가 -1%에 매수 주문
  ├── 현재가 +1%에 매도 주문
  ├── 현재가 -2%에 매수 주문
  ├── 현재가 +2%에 매도 주문
  └── ... (격자형 배치)
       ↓
  기존 주문 전부 취소 → 새 주문 일괄 등록
       ↓
  잔액 부족 시 자동 충전 (Faucet)
```

**왜 필요한가?**: 거래소에 주문이 없으면 사용자가 거래할 수 없음.
실제 유저가 적은 초기 단계에서 유동성을 제공하는 역할.
