# Nasun 도메인 비유 사전

이 프로젝트에서 자주 등장하는 기술 개념들을 일상 비유로 정리한 참조 문서입니다.
설명할 때 이 비유들을 활용하되, 맥락에 맞게 변형해도 됩니다.

---

## 블록체인 기초

| 개념 | 비유 | 풀어쓰기 |
|------|------|----------|
| 블록체인 | 모든 사람이 볼 수 있는 은행 거래 내역서 | 한 번 기록되면 수정 불가능한 공개 장부. 모든 거래가 투명하게 기록됨 |
| 블록 | 거래 내역서의 한 페이지 | 여러 거래를 묶어서 한 번에 기록하는 단위 |
| 트랜잭션 (TX) | 은행 이체 한 건 | 블록체인에서 일어나는 하나의 작업 (송금, 교환, 계약 실행 등) |
| 가스비 (Gas Fee) | 택배 배송료 | 트랜잭션을 처리해달라고 네트워크에 내는 수수료 |
| 지갑 (Wallet) | 인터넷 뱅킹 앱 | 내 자산을 관리하고 거래할 수 있는 도구. 비밀번호(개인키)로 보호됨 |
| 개인키 (Private Key) | 금고 비밀번호 | 이걸 아는 사람만 내 자산을 움직일 수 있음. 절대 공유하면 안 됨 |
| Devnet | 장난감 돈으로 하는 연습 게임 | 진짜 돈이 아닌 테스트용 네트워크. 마음껏 실험 가능 |
| Faucet | 연습용 돈 나오는 ATM | Devnet에서 테스트용 토큰을 무료로 받는 곳 |

## Sui/Nasun 특화

| 개념 | 비유 | 풀어쓰기 |
|------|------|----------|
| Move 언어 | 블록체인 전용 계약서 작성 도구 | 스마트 컨트랙트를 만드는 프로그래밍 언어. "물건은 복사 불가, 소유자만 이동 가능"이 핵심 규칙 |
| 오브젝트 (Object) | 실물 자산 (부동산 등기권리증) | 블록체인 위의 모든 것(토큰, NFT, 계정)은 "물건"처럼 존재. 소유자가 있고, 복사 불가 |
| Shared Object | 공용 게시판 | 누구나 읽고 쓸 수 있지만, 동시 접근 시 순서가 정해짐 |
| AdminCap | 관리자 열쇠 | 이 오브젝트를 가진 사람만 관리자 기능(설정 변경, 업그레이드 등) 사용 가능 |
| UpgradeCap | 리모델링 허가증 | 이미 배포된 스마트 컨트랙트를 업그레이드할 수 있는 권한 |
| Nasun Network | 우리만의 동네 | Sui 블록체인을 복제해서 만든 우리 전용 네트워크 (Chain ID: 272218f1) |

## 인증 & 보안

| 개념 | 비유 | 풀어쓰기 |
|------|------|----------|
| zkLogin | 나이 확인 스티커 | 신분증 전체를 보여주지 않고 "성인입니다"만 증명하는 기술. Google 로그인으로 지갑을 만들되, Google에겐 지갑 정보가 안 보임 |
| ZK Proof | 마술사의 증명 | "나는 비밀을 알고 있다"를 비밀 자체를 공개하지 않고 증명하는 수학적 기법 |
| Salt API | 비밀 양념 | Google 계정과 지갑 주소를 연결하는 고유한 값. 이걸 알아야 같은 지갑을 복원 가능 |
| Ephemeral Keypair | 일회용 도장 | 한 번의 로그인 세션에서만 쓰고 버리는 임시 키. 보안 강화 |
| AES-256-GCM | 최고급 금고 잠금장치 | 군사 수준의 암호화. 비밀번호 없이는 절대 열 수 없음 |
| PBKDF2 | 비밀번호 강화제 | 비밀번호를 10만 번 섞어서 해커가 추측하기 어렵게 만드는 기법 |
| Rate Limiting | 출입 제한 | 비밀번호를 너무 많이 틀리면 일정 시간 잠금. 무차별 대입 공격 방지 |

## 거래 & DeFi

| 개념 | 비유 | 풀어쓰기 |
|------|------|----------|
| DEX | 사람 없는 환전소 | 중개인 없이 코드(스마트 컨트랙트)가 자동으로 거래를 매칭하는 거래소 |
| CLOB (Central Limit Order Book) | 주식 시장의 호가창 | "나는 100원에 사겠다" / "나는 110원에 팔겠다" 주문이 모이는 곳. DeepBook이 이 방식 |
| AMM | 자동 환율 계산기 | 풀(저수지)에 두 토큰을 넣어두고, 비율로 가격을 자동 계산. CLOB과 다른 방식 |
| Maker | 주문을 걸어놓는 사람 | "100원에 사겠다"고 미리 적어놓는 사람. 거래가 성사되길 기다림 |
| Taker | 걸린 주문을 가져가는 사람 | 이미 걸려있는 주문을 보고 즉시 거래하는 사람 |
| BalanceManager | 거래소 전용 통장 | DeepBook에서 거래하려면 먼저 돈을 이 통장에 입금해야 함 |
| Unified Margin | 종합통장 | NUSDC, NBTC 등 여러 자산을 하나의 계정으로 묶어 관리. 한 곳에서 Spot + Perp + Lending 사용 |
| Perpetual Futures (무기한 선물) | 만료일 없는 베팅 | 가격이 오를지(Long) 내릴지(Short)에 레버리지(배율)를 걸어 베팅. 만료일 없음 |
| Leverage (레버리지) | 대출받아 투자 | 5x = 내 돈 100원으로 500원어치 포지션. 수익도 5배, 손실도 5배 |
| Liquidation (청산) | 강제 퇴장 | 손실이 너무 커지면 시스템이 자동으로 포지션을 강제 종료. 담보금 보호 |
| Funding Rate | 시소 균형추 | Long/Short 비율이 치우치면 수수료로 균형을 맞춤. 8시간마다 정산 |
| Oracle | 외부 시세 알리미 | 외부(실제 시장)의 가격 정보를 블록체인 안으로 가져오는 장치 |
| LP Bot | 자동 호가 로봇 | 바이낸스 가격을 참고해서 자동으로 매수/매도 주문을 걸어주는 봇 |

## Baram (AI Settlement)

| 개념 | 비유 | 풀어쓰기 |
|------|------|----------|
| TEE (Trusted Execution Environment) | 투명 금고 안의 계산기 | 암호화된 격리 공간에서 AI가 작동. 결과만 보이고, 입력 데이터는 아무도 (서버 관리자도) 볼 수 없음 |
| AWS Nitro Enclave | Amazon이 만든 투명 금고 | AWS에서 제공하는 TEE 기술. 물리적으로 격리된 환경 |
| Attestation (증명) | 봉인 확인서 | "이 금고가 변조되지 않았다"는 것을 암호학적으로 증명하는 문서 (PCR 값) |
| Escrow (에스크로) | 중립 보관함 | 거래 양쪽이 돈을 맡겨두고, 조건 충족 시 자동으로 정산되는 장치 |
| Budget Delegation | 용돈 위임 | AI 에이전트에게 "이 범위 안에서 알아서 써"라고 한도를 정해주는 것 |

## 웹 개발 기초

| 개념 | 비유 | 풀어쓰기 |
|------|------|----------|
| React | 레고 조립 시스템 | 화면을 작은 부품(컴포넌트)으로 나눠서 조립하는 방식. 부품 하나가 바뀌면 그것만 다시 그림 |
| React Hook | 센서/감지기 | 상태 변화를 감지하고 화면을 자동으로 업데이트하는 장치. useState, useEffect 등 |
| useState | 기억 칸 | 컴포넌트가 기억해야 할 값을 저장하는 칸. 값이 바뀌면 화면이 다시 그려짐 |
| useEffect | 자동 반응기 | "~가 바뀌면 ~를 해라"는 규칙을 설정. 데이터 가져오기, 타이머 설정 등에 사용 |
| useMemo | 계산 결과 캐시 | 복잡한 계산의 결과를 저장해두고, 입력이 바뀔 때만 다시 계산 |
| TanStack Query | 데이터 비서 | 서버에서 데이터 가져오기, 캐싱, 재시도를 자동으로 관리하는 도구 |
| Zustand | 공유 메모장 | 여러 컴포넌트가 함께 쓰는 전역 상태 저장소. 한 곳에서 바꾸면 다른 곳도 자동 반영 |
| WebSocket | 전화 통화 | 서버와 한 번 연결하면 실시간으로 양방향 대화 가능 (채팅, 실시간 가격 등에 사용) |
| REST API | 편지 주고받기 | 요청을 보내고 → 응답을 받고 → 연결 끊김. 한 번에 하나씩 처리 |
| localStorage | 브라우저 메모장 | 브라우저에 적어두는 임시 기억. 새로고침해도 유지되지만, 다른 기기에선 안 보임 |
| Vite | 초고속 요리 도구 | 개발 중에는 즉시 미리보기, 배포 시에는 최적화된 완제품으로 포장 |
| TypeScript | 맞춤법 검사기 | JavaScript에 "타입"을 추가해서 실수를 미리 잡아주는 도구 |
| Tailwind CSS | 스타일 스티커 | 미리 만들어진 디자인 조각을 붙여서 화면을 꾸미는 방식 |

## 인프라 & 배포

| 개념 | 비유 | 풀어쓰기 |
|------|------|----------|
| 모노레포 (Monorepo) | 아파트 단지 | 여러 앱(동)이 하나의 부지(저장소)에 있고, 놀이터/주차장(공유 패키지)을 함께 씀 |
| EC2 | 임대 서버 컴퓨터 | Amazon에서 빌린 가상 컴퓨터. 우리 앱이 여기서 돌아감 |
| Nginx | 안내 데스크 | 사용자 요청이 들어오면 올바른 앱으로 안내해주는 관문 |
| PM2 | 자동 복구 매니저 | 프로그램이 죽으면 자동으로 다시 살려주는 관리자 |
| rsync | 파일 동기화 | 변경된 파일만 골라서 서버에 업로드하는 도구 |
| CI/CD | 자동 배포 라인 | 코드를 올리면 자동으로 테스트 → 빌드 → 배포. (현재 Nasun은 수동 배포) |
| pnpm | 절약형 택배 창고 | 같은 패키지를 여러 앱이 공유해서 저장 공간과 설치 시간 절약 |
