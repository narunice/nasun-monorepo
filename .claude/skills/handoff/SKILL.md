# Session Handoff

세션 간 컨텍스트를 보존하는 핸드오프 문서를 생성/로드합니다.
컨텍스트 윈도우가 소진되기 전에 작업 상태를 저장하여 다음 세션에서 즉시 이어서 작업할 수 있습니다.

## 모드 선택

**핸드오프 생성**: 현재 작업 상태를 저장하고 싶을 때 → CREATE 워크플로
**핸드오프 로드**: 이전 작업을 이어서 할 때 → RESUME 워크플로

## CREATE 워크플로

### Step 1: 현재 상태 수집

아래 정보를 **병렬로** 수집합니다:

- `git status` — 변경된 파일 목록
- `git log --oneline -10` — 최근 커밋
- `git branch --show-current` — 현재 브랜치
- 현재 진행 중인 TodoWrite 목록

### Step 2: 핸드오프 문서 생성

`.claude/handoffs/` 디렉토리에 아래 형식으로 파일을 생성합니다:

**파일명**: `YYYY-MM-DD-slug.md` (예: `2026-02-13-prediction-seed.md`)

**템플릿**:

```markdown
# Handoff: [작업 제목]

**생성**: YYYY-MM-DD HH:MM
**브랜치**: [branch name]
**이전 핸드오프**: [있으면 링크, 없으면 "없음"]

## 현재 상태 요약

[2-3문장으로 현재 작업 상태 설명]

## 완료된 작업

- [x] 완료 항목 1
- [x] 완료 항목 2

## 미완료 작업

- [ ] 다음에 할 작업 1
- [ ] 다음에 할 작업 2

## 중요 컨텍스트

다음 세션에서 반드시 알아야 할 정보:

- **결정사항**: [내린 결정과 이유]
- **주의사항**: [알려진 이슈, 함정]
- **파일 위치**: [핵심 파일 경로]

## 최근 변경 파일

[git status/diff에서 추출한 변경 파일 목록]

## 즉시 다음 단계

1. [구체적이고 실행 가능한 첫 번째 액션]
2. [두 번째 액션]
```

### Step 3: 검증

- `[TODO]` 플레이스홀더가 남아있지 않은지 확인
- 비밀 정보(API 키, 비밀번호)가 포함되지 않았는지 확인
- 참조된 파일이 실제로 존재하는지 확인

### Step 4: auto memory 연계

핸드오프 작성 중 반복되는 패턴이나 교훈을 발견하면 `MEMORY.md`에 기록합니다.

## RESUME 워크플로

### Step 1: 핸드오프 목록 확인

```bash
ls -la .claude/handoffs/
```

### Step 2: 핸드오프 로드

가장 최근 핸드오프 또는 사용자가 지정한 핸드오프를 읽습니다.

### Step 3: 신선도 확인

핸드오프 생성 이후 변경사항을 확인합니다:

- `git log --oneline --since="[핸드오프 생성일]"` — 이후 커밋
- `git status` — 현재 워킹 트리 상태
- 핸드오프에 참조된 파일이 여전히 존재하는지 확인

### Step 4: 작업 재개

핸드오프의 "즉시 다음 단계" 항목 #1부터 시작합니다.

## 프로액티브 제안

5개 이상의 파일을 수정했거나, 복잡한 디버깅/아키텍처 결정을 거친 후:

> "상당한 진전이 있었습니다. 컨텍스트를 보존하기 위해 핸드오프 문서를 생성하시겠습니까? `/handoff`로 호출하세요."
