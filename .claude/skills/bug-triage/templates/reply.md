# Reply Templates

**Rules (strict):**
- Use "We" when referring to the ops team.
- Do not use em dash (—). Use commas, periods, parentheses, or hyphens instead.
- English only. Keep it short and warm.
- Thank the reporter; describe what we did; mention points granted.

## fixed (bug confirmed and shipped)

```
Thanks for the detailed report. We reproduced the issue and shipped a fix in the latest release. {one-line root cause summary}. We're awarding you {N} ecosystem points for helping us catch this.
```

## accepted (feedback / feature request we agree with)

```
Thanks for the thoughtful suggestion. We agree and have added this to our roadmap. {one-line plan or next step}. We're awarding you {N} ecosystem points for the contribution.
```

## duplicate (already reported, but still credit)

```
Thanks for flagging this. We had received a similar report and are tracking the fix under an existing item. {one-line status}. We're awarding you {N} ecosystem points since your details helped us confirm the pattern.
```

## cannot-reproduce (not enough info / cannot confirm)

```
Thanks for the report. We tried to reproduce this but could not confirm the issue on our side. If you hit it again, please share the browser, wallet address, and steps so we can dig deeper. We're awarding you {N} ecosystem points for the effort.
```

## wont-fix (valid observation but intentional behavior)

```
Thanks for writing this up. The behavior you describe is actually intentional for {brief reason}. We appreciate you taking the time to surface it, and we're awarding you {N} ecosystem points for the careful observation.
```
